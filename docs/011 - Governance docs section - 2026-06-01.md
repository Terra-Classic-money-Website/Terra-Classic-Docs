# Governance docs section

Created: 2026-06-01

## Purpose

Create a dedicated Governance section for Terra Classic Docs after the Forex Protocol section.

The section is for LUNC holders, USTC holders, delegators, community members, investors, and users who need to understand how Terra Classic governance works and how to participate responsibly. It is not primarily a developer reference or validator operations manual.

## Information architecture decision

Governance should be a top-level docs section because staking only explains how voting power is created. Governance explains how that power is used, how proposals move through the chain, how users vote, and how delegators hold validators accountable.

The new section owns:

- governance overview
- voting power and delegation
- proposal lifecycle
- vote options
- how to vote
- proposal evaluation
- validator accountability
- governance risks and safeguards

The Staking Protocol section should link to Governance where governance behavior is the main topic. Staking should keep validator selection and staking risk context, but not remain the canonical governance guide.

The implementation also follows the current docs architecture:

- markdown pages in `content/docs`
- navigation controlled by `scripts/build-docs-index.mjs`
- generated search/nav/manifest files regenerated by `npm run docs:index`
- validation through `npm run docs:validate`
