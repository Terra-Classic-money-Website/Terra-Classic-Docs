# Terra Classic Docs

Terra Classic Docs is an independent, community-maintained documentation site for Terra Classic.

The site is published at [docs.terra-classic.money](https://docs.terra-classic.money) and is designed to help users, delegators, validators, builders, governance participants, and external readers understand Terra Classic through clear, source-aware documentation.

## What This Site Covers

Terra Classic Docs organizes information about the Terra Classic network, including:

- core Terra Classic concepts and user guidance
- wallets, fees, staking, delegation, and governance
- validator and full-node operation
- developer references, smart contracts, endpoints, and tooling
- protocol documentation for staking, swaps, and proposed stable-asset systems
- institutional integration context, risks, and due-diligence material

The goal is to make Terra Classic easier to understand without turning documentation into promotion, price commentary, or validator-controlled messaging.

## Principles

This documentation project is built around a few simple principles:

- **Neutrality:** The docs should not favor validators, projects, exchanges, or private interests.
- **Source awareness:** Important claims should be tied to public sources, chain behavior, governance records, or repository history where possible.
- **Status clarity:** Live, proposed, historical, experimental, deprecated, and unverified material should be clearly distinguished.
- **Public contribution:** Corrections and improvements should be proposed openly through GitHub issues and pull requests.
- **User safety:** The docs should explain risks, limitations, and operational trade-offs instead of hiding them.

## What This Site Is Not

Terra Classic Docs is not a wallet, exchange, custody product, launchpad, validator, or financial-advice source.

It does not provide investment recommendations and does not replace a user's own verification before signing transactions, delegating tokens, running infrastructure, or relying on third-party services.

## Contributing

Community contributions are welcome.

Use GitHub issues to report errors, outdated information, unclear explanations, broken links, or missing topics. Use pull requests to propose corrections, new pages, source updates, and documentation improvements.

Good contributions are factual, specific, source-aware, and clear about whether a feature or claim is live, proposed, historical, or still under review.
