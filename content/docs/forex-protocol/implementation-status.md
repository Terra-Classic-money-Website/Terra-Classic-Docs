Forex Protocol is documented as a governance-accepted product concept, not as a deployed production product.

## Status checklist

Before any Forex Protocol page describes production availability, the docs should confirm:

| Item | Required evidence |
| --- | --- |
| Implemented CSM | Code, contracts, or module implementation exists and is reviewable. |
| Test coverage | Testnet behavior, simulations, or integration tests are documented. |
| Oracle integration | Price feeds and fallback behavior are implemented and tested. |
| Collateral model | Supported collateral, custody assumptions, and accounting rules are final. |
| Minting flow | User or operator minting path is documented. |
| Redemption flow | Redemption path, fees, caps, and failure states are documented. |
| Security review | Audit, independent review, or explicit risk acceptance exists. |
| Mainnet launch | Governance or deployment evidence confirms production availability. |

Until those items exist, use proposed or governance-accepted language.
